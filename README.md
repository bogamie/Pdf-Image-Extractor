# PDF 이미지 추출기 (TUI)

PDF에서 **임베디드(래스터) 이미지**와 **벡터 그래픽**을 인식해 정확한 영역만 이미지로 추출하는 TUI(Terminal User Interface) 도구입니다.

## 실행

```bash
python main.py
```

## 주요 기능

- **키보드 조작**: vim 스타일의 직관적인 키보드 조작
- **실시간 미리보기**: 페이지 번호 입력 시 즉시 미리보기 표시
- **고품질 이미지 미리보기**: Ghostty, Kitty, WezTerm 등의 터미널에서 Kitty 그래픽 프로토콜을 이용한 실제 이미지 미리보기 지원 (다른 터미널은 ASCII 아트로 폴백)
- **자동 영역 인식**: 임베디드 이미지와 벡터 그래픽을 자동으로 감지하여 추출
- **스마트 병합**: 인접한 벡터 요소를 하나의 이미지로 자동 병합

## 키보드 조작

### 파일 탐색 모드
- **↑ / ↓**: 파일/폴더 이동
- **Enter**: 폴더 진입 또는 PDF 선택 (페이지 입력 모드로 전환)
- **q**: 상위 폴더로 이동 또는 종료 (기본 디렉터리까지 도달 시)

### 페이지 입력 모드
- **숫자 입력**: 페이지 번호 입력 (입력 즉시 미리보기 업데이트)
- **↑ / ↓**: 페이지 번호 ±1
- **Enter**: 현재 페이지의 이미지 추출
- **Backspace**: 입력한 숫자 삭제
- **q**: 파일 탐색 모드로 돌아가기

## 출력 정보

- **저장 위치**: `~/Pictures/Extracted Images/<PDF파일명>/`
- **파일명**: 
  - 이미지 1개: `{PDF이름}_{페이지}.png`
  - 여러 개: `{PDF이름}_{페이지}_1.png`, `{PDF이름}_{페이지}_2.png`, ...
- **기본 디렉터리**: `~/Documents`

pip install -r requirements.txt

## 지원하는 터미널

### Kitty 그래픽 프로토콜 지원 (실제 이미지 미리보기)
- **Ghostty** (권장)
- **Kitty**
- **WezTerm**

### ASCII 아트 폴백 (기타 터미널)
- 모든 터미널에서 작동하지만 ASCII 문자로 미리보기가 표시됩니다.

## 지원하는 이미지 종류

| 종류 | 설명 | 추출 방식 |
|------|------|------------|
| **임베디드 이미지** | PDF에 삽입된 비트맵 (JPEG, PNG 등) | 원본 바이너리 추출 (투명도/마스크 지원) |
| **벡터 그래픽** | 선, 도형, 차트, 다이어그램 등 | 영역 자동 인식 후 고해상도 PNG로 변환 |

## 추출 알고리즘

1. **임베디드 이미지**: PDF 내부의 XObject 이미지를 직접 추출
   - 투명도 마스크(SMask) 자동 처리
   - 원본 품질 유지

2. **벡터 그래픽**: 
   - 페이지의 모든 벡터 경로(path) 분석
   - 인접한 요소 자동 병합 (기본 25pt 이내)
   - 헤더/푸터/장식선 자동 필터링
   - 300 DPI로 래스터화하여 PNG 저장

## 요구 사항

- **Python**: 3.8 이상
- **PyMuPDF**: PDF 처리 및 이미지 추출 (`pip install pymupdf`)
- **curses**: TUI 인터페이스 (Linux/macOS 기본 포함, Windows는 `windows-curses` 필요)

## 사용 예시

1. 프로그램 실행:
   ```bash
   python main.py
   ```

2. `~/Documents` 폴더에서 PDF 파일 찾아 선택

3. 페이지 번호 입력 (예: `94`)

4. 미리보기 확인 후 Enter로 이미지 추출

5. 추출된 이미지는 `~/Pictures/Extracted Images/<PDF이름>/` 폴더에 저장됨

## 알려진 제한 사항

- 스캔된 PDF의 경우 전체 페이지가 하나의 이미지로 취급될 수 있습니다.
- 매우 복잡한 벡터 그래픽은 추출에 시간이 걸릴 수 있습니다.
- Kitty 그래픽 프로토콜은 지원되는 터미널에서만 작동합니다.
